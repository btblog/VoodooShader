Readme (v0.2.3.143)

Introduction

The Voodoo Shader Framework is a comprehensive support system for shader effects. It is designed to be used with a number of graphics API and programs, as well as game modding.

Voodoo is made up of two main parts. The core contains the shader loading and initialization algorithms, as well as some universal management features. Each game or program using Voodoo then provides an adapter, which handles the details of attaching the Voodoo core with a particular program's graphics system.

Prerequisites

The Voodoo Shader Framework requires a few additional files to run properly. These include:

  • Microsoft Visual C++ 2010 Runtime (x86)
  • Latest drivers for your video card

The Voodoo log viewer requires the Microsoft .Net framework 3.5 and Internet Explorer 5.0 or better installed.

Some Voodoo adapters may require OpenGL drivers or the DirectX 9.0c, 10 or 11 runtimes to be installed, as well as Voodoo plugins. These additional dependencies are indicated in the adapter documentation.

Installing the Core

The Voodoo Core comes with a small installation wizard. The wizard installs all the necessary files and creates the registry key with the Voodoo path. The wizard should easily handle all installation steps for the core.

The installer does not need administrative permission to run, but if run by a typical user, the Voodoo installation will only work for that user. Administrators can install a system-wide copy of the Voodoo core. Per-user copies will be preferred over system copies, if both are present.

The registry key is required for adapters to locate the core; it holds the current location of the Voodoo core (and may contain other data in the future). The other possible solutions were either complicated, dangerous for stability, messy or buggy. Using a registry key, while somewhat unpopular, is clean, safe and seemingly quite stable.

There is no automatic update utility at the moment, although one may be created in the future. Updating the core will require downloading and installing the latest files.

Installing Adapters

Once the Voodoo Core has been installed, additional adapters may be installed for any supported program you would like to use Voodoo with. Adapters may use install wizards or require manual installation. Some adapters may require a minimum version of the Voodoo core to function.

For adapters with installers, you should run the installer and follow the instructions. These will usually automatically detect the Voodoo core path and any other locations, then install the necessary files. Some adapters may require additional support files (the DirectX adapters require the latest DirectX runtimes).

For adapters without installers, you should extract the archive to a temporary folder and find the readme. This should contain instructions on how to install the adapter files. You may need to know the locations of the Voodoo core and other programs on your system, and should have permission to add files to those locations. Most Voodoo adapter DLLs go in a bin folder inside the main Voodoo folder.

Running Voodoo

Once you have the Voodoo core and any chosen adapters installed, whenever you run supported program, Voodoo should be automatically used.

Shortcuts and other launcher applications should (depending on how they handle things) launch the program with Voodoo enabled. If you find a launcher which does not, or run into a problem, please report it immediately.

Uninstallation

To remove the Voodoo core, you should run the included uninstaller. This can be access from the Voodoo folder or through the start menu (if you chose to add the start menu shortcuts).

To uninstall Voodoo adapters, you should run the adapter-specific uninstaller (if the adapter used an installation wizard) or follow any instructions in the readme.

To temporarily disable Voodoo for a specific program, you should usually remove or rename the hook DLL (for the DirectX 8.9 and 9.0 hooks, this is a small file named d3d8.dll or d3d9.dll). This will disable Voodoo for this program until you restore the hook.
