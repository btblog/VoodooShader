Detailed Description

Contains a hardware texture, for use with Voodoo effects.

These textures are usually bound to parameters and shaders can sample pixels from or draw to them to create the desired effects. Textures may also represent a depth texture, with slightly different handling, or a null texture with no data storage. Textures may be created by calling ICore::CreateTexture() or ICore::LoadTexture(), depending on the source. Textures are provided by the binding.

IID:

Note:
Textures have ITexture::StretchRect and ITexture::CopyVolume for transferring data. There is no specific method for 1D textures, ITexture::StretchRect() should be used in this case, and the Y component set to 1 (other values will cause undefined behavior).

virtual VoodooResult VoodooShader::ITexture::Bind ( TextureMode  mode,
uint32_t  index 
) [virtual]

Binds this texture to a render-target on underlying API.

This can only be done with textures that were created with VSTexFlag_Target, other textures must fail regardless of API support.

Parameters:
modeThe binding mode for this texture, whether it will be used as a source or target.
indexThe index to be bound to.
Note:
Every call to ITexture::Bind should have a corresponding call to ITexture::Reset.
Warning:
Binding a texture as both source and target is typically not possible with the hardware API. To allow this in most circumstances, Voodoo Shader may do some indirection within the binding. This is very likely to have a performance penalty, so it should be avoided.
Note:
If mode is VSTexMode_Target, index may be VOODOO_TEXTURE_DEPTH. This will bind the texture to the depth buffer. For this to work, the texture must be a depth format (VSFmt_D16, VSFmt_D24, VSFmt_D24S8, VSFmt_D32, or VSFmt_DMax). In some cases, these textures may also be bound as VSTexMode_Source.

virtual VoodooResult VoodooShader::ITexture::StretchRect ( ITexture *  pSource,
const Rect  source,
const Rect  dest 
) [virtual]

Copies a 1 or 2D region of texture data from another texture.

Parameters:
pSourceThe texture to copy from. This must be the same format with the same mipmap flags.
sourceThe source rectangle.
destThe destination rectangle.
Note:
This method may do any stretching of the texture data. However, stretching may limit the possible allowed sources and destinations.
If pSource has more dimensions than this texture, any more than this texture has must be disregarded. For example, if pSource is 3D and this is 2D, only the first "slice" will be copied. If pSource has fewer dimensions, the copy must fail. If any other behavior is needed, some form of render-to-texture should be used.
Warning:
Under some APIs, this call may require extra handling to copy from render target textures to others; this may result in a notable performance hit. Because of this, avoid copying from render target textures.
