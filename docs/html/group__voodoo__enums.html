Detailed Description

Texture formats for use by textures, describing the layout and size of the texture data.

These may not be implemented by the underlying graphics API exactly as they are indicated here, but the available components and sizes are guaranteed to be equal to or greater than the indicated values (with the exception of VSFmt_Null and VSFmt_DMax, which have no and variable precision). Further information on texture formats may be found on the texture formats chart.

Many bindings restrict the use of depth textures as inputs to effects. The listed formats are mostly compatible for use as such, but this is still very hardware-dependent. See Depth Buffers for details.

enum VoodooShader::CompilerProfile

Target compiler profiles.

These control the compiler used for Voodoo Shader resources.

Note:
Various portions of this enum are reserved or have a special purpose:
  • 0x00FF: Version within the API. Typically this matches the version the API itself reports.
  • 0x7FF0: Major API or vendor. The unset API is reserved for internal Voodoo Shader profiles.
  • 0x8000: Debug flag, where the API provides a debug version.

enum VoodooShader::LogLevel

Log message levels.

These are set up to quickly filter messages based on severity and source. Each message must have a severity and source bit set, which the logger tests against its internal filter, like so:

 LogLevel maskedLevel = level & storedLevel;
 if ( (maskedLevel & VSLog_Origin) && (maskedLevel & VSLog_Severity) )
 {
     // Log message
 }

enum VoodooShader::ParseFlags

String parsing flags.

These modify the behavior of the string parser.

Slash flags are handled before path flags. Path flags use system API calls to perform their processing, and are dependent on system behavior. More flags will be added when logic, functions and operators are added to variables.
