VoodooShader::VoodooWFS::VOODOO_CLASS ( VSWFileSystem  ,
IFileSystem

Provides a unified file management system for loading shaders and other resources.

This file system implementation is a thin wrapper for the Windows API, adding only a few nonstandard functions (directory searching and path variables).

CLSID
e6f312b1-05af-11e1-9e05-005056c00008

Add a directory to the search path. Directories are pushed to the front of the list, which is searched in order (last added has highest priority).

Note:
This function uses Parser::ParseString() for all paths. Variables are evaluated when the path is added.
This function will split paths at the ';' character, similar to how Windows treats the path variable. This (combined with env vars) allows regular-form environment vars to be added to the file system.

Removes a directory from the search path, following the same rules as adding (adding a string then removing the same string will always work).

Parameters:
pathThe path to remove.

Resolves a relative filename (usually just filename and extension, but may include directories) into an absolute filename by searching the list of resource directories registered with this manager.

Parameters:
nameThe filename to search for.
Returns:
If the file is found, a reference to an unopened file object is returned.
Note:
This functions uses Parser::ParseString() on the path. It is evaluated once when the function enters, then appended to each stored path until a valid file is found.

Creates a file object from a path (usually an absolute path).

Retrieves this file's absolute path.

Returns:
Internal path.

Opens the file for read-write access.

Parameters:
modeThe mode to open the file in.
Returns:
Whether the file was successfully opened.
Note:
If this file handle was returned by a VSWFileSystem and the absolute path is still valid, this function should always succeed. In single-threaded or fullscreen scenarios with few user tasks, this is usually the case.
Warning:
If this file handle uses a relative path, then it is subject to changes in working directory by the any module in the process. VSWFileSystem::GetFile() uses an absolute path in the constructor and is not susceptible to this.

Closes the file, preventing further access.

Returns:
Success of the operation

Reads a chunk of data from the file. The file must have been opened for reading previously, or this call will fail.

Parameters:
countThe number of bytes to read, -1 for all.
pBufferThe buffer to be read into (may be nullptr, see notes).
Returns:
The number of bytes read.
Note:
If
  • buffer is nullptr, the number of bytes that would have been read is returned but the file position is unchanged. If
  • count is -1, this returns the filesize remaining
Writes a chunk of data to the file. The file must have been opened for writing.
Parameters:
countThe number of bytes to write.
pBufferThe data to write.
Returns:
Success of the write operation.
Note:
If buffer is nullptr, count zeros are written into the file. This is useful for padding binary formats.

Provides image loading, using the DevIL library.

This class provides internal loading and conversion, and can manage 1-3 dimensional images (regular textures and volumes).

Retrieves texture format data from the image. The runtime can use this to set up a texture or decide if it can handle the texture format.

Returns:
Texture information.

Retrieves a portion of the texture data from the image.

Parameters:
pDescThe region and format to be returned.
sizeThe size of the buffer
pBufferThe memory for the return data to be placed in. Must already be allocated, or nullptr.
Returns:
The number of bytes retrieved (or, if buffer is nullptr, the number that would be retrieved).
Warning:
Due to limitations in this library (or DevIL, not sure which), the only texture formats this function can convert into are TF_RGB8, TF_RGBA8, TF_RGBA16F and TF_RGBA32F. Others are not supported and will cause this function to throw.
Note:
This can convert data between most formats, so the format given in desc will be the returned format. This makes calculating the buffer size relatively easy.
Warning:
If this function converts formats or copies a large region, it will be slow. Avoid calling often.
