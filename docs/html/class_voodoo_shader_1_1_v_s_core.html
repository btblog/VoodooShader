virtual VoodooResult VoodooShader::VSCore::AddEvent ( Uuid  event,
Functions::CallbackFunc  func 
) [virtual]

Registers a callback with this core for a particular event.

This allows plugins to subscribe to various core events, such as init finalization, reset, and so on. Not all of the events are directly triggered by the core, this acts as a centralized event processor for the entire framework.

Parameters:
eventThe Uuid of the event to subscribe to.
funcThe callback function for the event.
Warning:
It is not possible to type-check function pointers across module boundaries. All callbacks must meet the given function prototype exactly, or the library can crash. To help simplify this, the callback prototype takes a list of variants, allowing a variety of parameters while retaining safety (this will be changed to a map of variants in the future, once a map class is added).

virtual VoodooResult VoodooShader::VSCore::Bind ( CompilerProfile  profile,
uint32_t  count,
Variant *  pParams 
) [virtual]

Binds the core to a particular compiler profile.

This loads and initializes the compiler, attaching it to this core until the core is reset or destroyed. Compiler profiles interface the Voodoo Shader core with a given hardware API, version, or profile.

Parameters:
profileThe compiler to use.
countThe number of parameters passed.
pParamsParameters for the binding.
Note:
The number and contents of pParams are compiler-defined. Most standard compilers take a single parameter, with the device or context to be used.
Precondition:
ICore::Init()
Postcondition:
If this call succeeds, resources may be created from this core.

uint32_t  count,
Variant *  pArgs 
) [virtual]

Broadcasts an event to all registered handlers, passing the provided argument list.

Parameters:
eventThe Uuid of the event to call.
countThe number of variants passed, may be 0.
pArgsThe list of variants passed as arguments, may be nullptr if count is 0.
Warning:
This calls all registered callbacks for the given event. The order registered callbacks will be called in is undefined.
Callbacks can modify the arguments, but not change the count.
If a callback returns a failure value, the loop will be aborted and remaining callbacks will nto be called. The failure will be returned by this method. If no callbacks fail, this will return generic success.

ParameterDesc  desc 
) [virtual]

Creates a global virtual parameter.

This parameter exists in the core, but is not a part of any effect.

This is most useful for creating parameters that must be shared between programs. Only parameters created with this function may be used in Parameter::Link().

Parameters:
nameThe name for this parameter.
descThe desc of the parameter to create.
Precondition:
ICore::Bind must be called successfully before use.
Note:
This function creates global parameters, which are not associated with any effect. They are primarily used to propagate values down to other parameters, both global and effect.

virtual VoodooResult VoodooShader::VSCore::Reset ( ) [virtual]

Cleans up all modules and objects.

Invalidates all objects created by this core. This method provides an early cleanup (in cases where process-termination cleanup may cause crashes). If successful, this core may be reinitialized.

Returns:
Success of the destroy operation.
Warning:
If this method fails, this core is in an undefined state and must be destroyed; calling any method but AddRef or Release is undefined. All resources created by this core are immediately considered invalid.
Precondition:
ICore::Init()
Postcondition:
If this call fails, the core is in an undefined state and must be destroyed.
