Detailed Description

Voodoo Shader null adapter implementation.

Requires no graphics API and does not call core methods, simply returning true or nullptr for methods. Does not perform logging or parameter validation.

Performs type checking internally, then provides a pointer to this object with the desired interface if possible.

Parameters:
clsidThe interface Uuid desired.
ppOutA pointer to the new interface, passed as void**.
Returns:
Success of the cast or uuid-fill.
Note:
If ppOut is nullptr and clsid is a null Uuid , clsid will be filled with the this class' CLSID.
If ppOut is nullptr and clsid is not a null Uuid, this will simply check if the Uuid is allowed without performing a cast.

virtual VoodooShader::VSAdapter::__declspec ( nothrow  ) const [virtual]

Loads a pass in a manner that is compatible with the adapter's underlying API.

For Cg-supported APIs, this may use the appropriate module loading function (cgD3D9LoadProgram, cgGLLoadProgram, etc). This may involve additional compilation or linking. In some cases, the pass may not be specifically loaded and this may simply return true.

Parameters:
pPassThe pass to be loaded.
Returns:
Whether or not the pass was successfully loaded.
Warning:
There is no guarantee what programs, if any, will be loaded or how this will be performed. The only guarantee is that if this call returns true, the pass may be bound to this adapter in the future. If this returns false, the pass must not be bound.
